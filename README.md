# AI EPUB 翻译助手

这是一个EPUB翻译工具，能够翻译内容时保留原文件的所有排版、图片和文字样式。

---

### 1. 获取与运行方法

#### 方式 1：Windows 直接运行 (推荐)
1. **下载release压缩包**：前往 GitHub [Releases](https://github.com/31naitnij/AI-EPUB-Translator/releases) 页面。
2. **解压缩并运行**

#### 方式 2：自行构建 (适用于开发者)
1. **准备环境**：建议使用 Python 3.10+。
2. **克隆仓库**：`git clone https://github.com/31naitnij/AI-EPUB-Translator.git`
3. **安装依赖**：`pip install -r requirements.txt`
4. **执行构建**：
   - **Windows**: 双击运行 `build_exe.bat` 或执行 `python build.py`
   - **macOS**: 运行 `bash build_macos.sh` 或执行 `python3 build.py`
   - **Linux**: 执行 `python3 build.py`
5. **获取结果**：生成的可执行文件将位于 `dist/` 文件夹内。

---

### 2. 使用说明

- **顶部配置**：设置 EPUB 文件、API URL/Key 及模型名称。
- **任务列表**：点击"分块并分组"后，支持 Ctrl/Shift 多选任务进行批量处理。
- **编辑回填**：在右侧对照区修改译文后，点击"保存组修改"即可即时验证并回填至导出镜像。

---

### 3. 双语对照与纯译文双版本输出

点击"导出"按钮后，程序会自动生成**两个版本**：

| 输出文件 | 说明 |
| :--- | :--- |
| `translated_{书名}.epub` | **纯译文版** — 原文被译文替换 |
| `bilingual_{书名}.epub` | **双语对照版** — 原文在上、译文在下交替排列 |

**对照版示例**：
```html
<p>The world is full of magic things, patiently waiting for our senses to grow sharper.</p>
<p>世界上充满了神奇的事物，正耐心地等待我们的感官变得更加敏锐。</p>
```

> **实现原理**：首次解压 EPUB 时会创建一份永不修改的原始母版 (`original/`)，每次导出时从母版复制干净副本，分别进行"替换式回填"和"插入式回填"，确保可重复输出。

---

### 4. 核心原理：锚点系统

为了让 AI 不受 HTML 复杂代码干扰，我们将文本抽象为以下标记：
- `<n>...</n>`：**块级标记**，用于定位段落或标题。
- `<tn>...</tn>`：**内部格式标记**，用于保留加粗、斜体、链接等行内样式。
- `<sn/>`：**自闭合标记**，用于保留图片、换行等位置。

AI 仅需翻译文字并保留这些标记的原有位置，程序即可精准还原所有排版。

---

### 5. 格式错误与手动修复

当任务显示为 **红色** 时，代表格式校验未通过，该部分内容暂时不会回填到最终文档中。

| 错误类型 | 原因 | 处理方法 |
| :--- | :--- | :--- |
| **块数不匹配** | AI 合并或截断了内容 | 手动补全 `<n>` 标记，确保块数量与原文一致。 |
| **内部标签损坏** | `<tn>` 标签未闭合或编号变动 | 检查并修复成对的 `<tn>` 标签。 |
| **自闭合标签缺失** | `<sn/>` 被 AI 遗漏 | 补齐缺失的 `<sn/>` 标签。 |

**手动修复流程**：在右侧编辑器修改 -> 点击"保存组修改" -> 若颜色变回正常（透明/白色），则表示回填成功。

---

### 6. AI 提示词建议

建议在 Prompt 中包含以下限制，以获得最佳格式稳定性：

```text
你是翻译专家，把英文翻译成中文。
规则：
1. 严格保留所有数字标签（如 <1>、<t1>、<s1/>）及其编号与闭合，不合并或拆分块。
2. 直接输出翻译结果。

示例：
<1>Chapter 1: <t1>The</t1> Begins</1>
<2>Text with <t1>styles</t1><s1/>.</2>
->
<1>第一章：<t1>开始</t1></1>
<2>带有<t1>样式</t1><s1/>的文本。</2>
```

---

### 7. 注意事项

- **文件占用**：翻译及导出前请关闭阅读器中的该文件，以免写入失败。
- **隐私提醒**：`config.json` 包含私密 API Key，请勿通过任何方式公开分发。
- **旧缓存兼容**：如果你有之前版本生成的缓存（没有 `original/` 目录），需先"清除缓存"再重新"分块并分组"。
