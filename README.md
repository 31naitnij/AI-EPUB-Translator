# AI 文档翻译助手 (EPUB/DOCX)

一款基于大模型 (LLM) API 的智能文档翻译工具。  
**核心目标**：提供出版级精度的翻译体验，实现 **100% 保留**原文档的样式、排版、图片、表格、脚注及目录结构。

---

## ✨ 核心亮点

### 1. 🛡️ 排版零损坏 (Anchor-Following)
传统翻译工具常破坏文档结构。本工具采用自研的 **“锚点跟随”** 技术：
- 将所有格式标签（加粗、斜体、图片、链接）替换为唯一锚点（如 `⦗1⦘`）。
- 仅将纯文本发送给 AI 翻译。
- 译后根据锚点ID精准还原格式。
**结果**：译文的内容变了，但样子和原文一模一样。

### 2. 🪞 实时源码镜像 (Live Source Mirror)
您的每一次翻译（包括自动翻译和手动修改）都会**实时**写入到硬盘上的源码镜像中。
- **随时中断**：不用担心进度丢失，随时关机，下次接着翻。
- **混合输出**：支持只翻译特定章节。导出的文档将是“译文+原文”的完美混合体，未翻译部分保持原样。

### 3. 🧩 智能边界分块
- 自动按段落切分，并合并为最佳大小（Chunk Size）发送给 API。
- **严格遵守文件边界**：EPUB 模式下，翻译组绝不跨越 HTML 文件边界，确保章节独立性和上下文安全。

### 4. 📝 全能手动干预
- 觉得 AI 翻得不好？在界面右侧直接修改译文。
- 点击保存，修改立即生效并同步到最终导出文件中。

### 方式一：直接运行 (Windows)
1. 下载最新 Release 版本。
2. 双击 **`run.bat`** 即可启动。

需要 Python 3.10+ 环境。
1. 安装依赖：`pip install -r requirements.txt`
2. 运行程序：`python main.py`

## � 使用步骤指南

### Step 1: 准备文件
打开软件，点击左上角 **“选择文件”**，加载您的 `.epub` 或 `.docx` 书籍。
*(建议提前确认书籍没有加密 DRM)*

### Step 2: API 配置
在顶部栏填入您的 LLM API 信息：
- **Key / URL / Model**: 支持 OpenAI, DeepSeek, Claude 等任何兼容 OpenAI 格式的 API。
- **温度 (Temperature)**: 建议 **0.3 - 0.7**。非虚构类书籍建议低一点。
- **分块大小**: 建议 **1000 - 2000** 字符。
- **上下文轮数**: 建议 **1**。让 AI 参考上一组的译文，保持人名地名一致。

### Step 3: 初始化
点击底部 **“分块并分组”**。
程序会解析全书，并在左侧列表中生成任务队列。

### Step 4: 开始翻译
- **全书翻译**：点击 **“开始翻译”**。
- **局部翻译**：在左侧列表中按住 `Shift` 或 `Ctrl` 选中部分行，点击 **“翻译选中组”**。

### Step 5: 导出
等待翻译完成（或随时停止），点击 **“导出”**。
程序会生成一个新的文件（如 `book_translated.epub`），保持原书所有细节。

## 🤖 推荐提示词 (Prompt Example)

本工具依赖 Prompt 来让 AI 理解“锚点”规则。  
软件默认内置了 Prompt，但您也可以根据需要微调。以下是一个经过验证的**最佳实践 Prompt**：

> [!TIP]
> 请直接复制以下内容到软件的 Prompt 设置框中：

```text
你是一个精通多国语言的专业翻译引擎。你的任务是将给定的文本片段从【源语言】翻译成【目标语言（中文）】。

### 关键规则（必须严格遵守）：
1. **保留锚点**：文本中包含形如 `⦗n⦘`、`⟬...⟭`、`⟦...⟧` 的特殊标记，这些是由于排版格式（如加粗、链接、注脚）转换而来的锚点。
   - ⚠️ **绝对禁止** 删除、移动、修改或重新编号这些锚点。
   - 锚点必须紧随其原本修饰的文字。

2. **分隔符**：输入文本中包含 `⧖` 符号作为逻辑块的分隔。
   - 输出时必须保留所有 `⧖` 符号，且位置与原文一一对应。

3. **翻译策略**：
   - 只翻译文本内容。
   - 保持原文的语气和风格。
   - 专有名词（人名、地名）如果通常保留原文，则保留；否则请提供准确译名。

### 输入输出示例：

**输入**：
⧖Hello ⦗1⦘world⦗2⦘!⧖How are ⟦you⟧ today?⧖

**输出**：
⧖你好 ⦗1⦘世界⦗2⦘！⧖你今天⟦怎么样⟧？⧖
```

## ❓ 常见问题 (Q&A)

**Q1: 导出时提示“Permission denied”或没有反应？**
> **A**: 请检查您是否正在用阅读器（Kindle PC, Calibre, Word 等）打开该文件或输出目录中的同名文件。**Windows 对文件占用极其严格，请务必关闭所有相关软件后再试。**

**Q2: 为什么有的段落还是英文？**
> **A**: 
> 1. 您可能选择了“翻译选中组”，未选中的部分自然保持原文。
> 2. AI 可能在该段落响应超时或格式错误，触发了程序的**自动容错机制**（回退到原文以保护文档结构）。您可以手动选中该段落重新翻译。

**Q3: 翻译速度太慢？**
> **A**: 速度完全取决于 API 的响应速度。您可以尝试增加“分块大小”(Chunk Size)，减少网络请求次数（但也别太大，以免 AI 丢失细节）。

**Q4: 支持 PDF 吗？**
> **A**: 不直接支持。建议先用 Word 或 Adobe Acrobat 将 PDF 转为 DOCX，再使用本工具翻译，效果最好。
