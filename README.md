# AI EPUB 翻译助手

这是一个EPUB翻译工具，能够翻译内容时保留原文件的所有排版、图片和文字样式。
最新版本支持“默认模式”与全新的“直接 HTML 模式”，为不同场景的翻译需求提供极大的灵活性。

---

### 1. 翻译模式介绍

系统提供两种翻译模式，可在运行界面的下拉框或控制台脚本中切换：

#### 模式 A: 默认模式 (标签简化 + 双语输出)
- **原理**：将复杂的 HTML 标签简化为类似 `<1>`, `<t1>`, `<s1/>` 的数字锚点，从而让 AI 的注意力完全集中在文本上，降低幻觉概率。
- **输出**：在导出时，会自动生成**纯译文版**和**双语对照版**两个版本的 EPUB。
- **格式校验**：系统会对 AI 的输出进行严格的标号对称性校验，防止丢段、错位。

#### 模式 B: 直接 HTML 模式 (全文本 + 仅纯译文)
- **原理**：不对任何标签进行简化，提取完整的原生 HTML 行发送给 AI。AI 需要直接返回带有原生 HTML 标签的翻译结果。
- **输出**：仅生成一份完美的**纯译文版** EPUB，保留原生 HTML 排版，无双语对照功能。
- **特点**：去除了所有强制的格式校验逻辑，不会产生由于标签不对称导致的红色报错。适合模型能力极强且有特殊排版定制需求的场景。

### 2. 获取与运行方法

#### 方式 1：Windows 直接运行 (推荐)
1. **下载 release 压缩包**：前往 GitHub [Releases](https://github.com/31naitnij/AI-EPUB-Translator/releases) 页面。
2. **双击启动**：解压后双击打开 EXE 文件即可。

#### 方式 2：使用启动脚本 (Windows)
1. **准备环境**：安装 Python 3.10+。
2. 双击运行 `run.bat`，在控制台中选择 1 (默认模式) 或 2 (直接 HTML 模式)，系统会自动构建虚拟环境并安装依赖。

#### 方式 3：自行构建 (适用于开发者)
1. **准备环境**：建议使用 Python 3.10+。
2. **克隆仓库**：`git clone https://github.com/31naitnij/AI-EPUB-Translator.git`
3. **安装依赖**：`pip install -r requirements.txt`
4. **执行构建**：
   - **Windows**: 双击运行 `build_exe.bat` 或执行 `python build.py`
   - **macOS**: 运行 `bash build_macos.sh` 或执行 `python3 build.py`
   - **Linux**: 执行 `python3 build.py`
5. **获取结果**：生成的可执行文件将位于 `dist/` 文件夹内。

### 3. 使用说明

- **顶部配置**：选择翻译模式，设置 EPUB 文件路径、API URL/Key 及模型名称。
- **预设提示词**：切换翻译模式时，提示词输入框会自动加载适配当前模式的 Prompt 规则。
- **任务列表**：点击"分块并分组"后，支持 Ctrl/Shift 多选任务进行批量翻译。
- **编辑回填**：在右侧对照区修改译文后，点击"保存组修改"即可即时验证并写入到缓存中。
- **最终导出**：点击"导出"按钮生成翻译完成的 EPUB 文档。

### 4. 常见问题处理 (默认模式下)

在**默认模式**中，当任务显示为 **红色** 时，代表格式校验未通过，该部分内容暂时不会回填到最终文档中：

| 错误类型 | 原因 | 处理方法 |
| :--- | :--- | :--- |
| **块数不匹配** | AI 合并或截断了内容 | 手动补全 `<n>` 标记，确保块数量与原文一致。 |
| **内部标签损坏** | `<tn>` 标签未闭合或编号变动 | 检查并修复成对的 `<tn>` 标签。 |
| **自闭合标签缺失** | `<sn/>` 被 AI 遗漏 | 补齐缺失的 `<sn/>` 标签。 |

**手动修复流程**：在右侧编辑器修改 -> 点击"保存组修改" -> 若颜色变回正常，则表示格式验证通过并回填成功。

*(注：在**直接 HTML 模式**中，格式检查已被禁用，不会出现红色报错，用户只需确保生成的 HTML 结构没有被严重破坏即可。)*

---

### 5. 注意事项

- **文件占用**：翻译及导出前请关闭阅读器中打开的对应书籍，以免因文件被占用导致写入失败。
- **隐私提醒**：`config.json` 包含私密 API Key 和本地路径信息，请绝对不要将其上传至公开仓库或分享给他人。
- **旧缓存兼容**：如果你有之前版本生成的单文件缓存，请先在工具界面点击"清除缓存"，再重新进行"分块并分组"。
